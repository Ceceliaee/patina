# 外部导入集成收尾执行方案

状态：已完成并归档

关联事项：GitHub Project「建立通用时间记录导入能力」

## 第一性原则

- [x] Patina 原生记录与外部导入记录必须保持物理身份隔离，任何维护操作都不能让二者互转。
- [x] 原生记录在时间重叠时拥有统计优先级，外部数据只补充原生记录未覆盖的时间。
- [x] 只有具备精确起止时间的记录进入时间线；小时汇总只进入汇总、排行和图表。
- [x] 分类页对原生和外部记录提供相同的按应用管理体验；导入弹窗额外提供按批次删除能力。
- [x] 读模型必须在导入数据变更后失效，界面不能继续展示已经删除的旧快照。
- [x] 所有文案必须准确说明操作影响的记录所有权与范围。

## 一、导入批次删除后的刷新

- [x] 识别 Dashboard、History、Data、Tools 与分类读模型使用的缓存边界。
- [x] 在批次删除成功后统一清理受影响缓存，并触发必要的读模型刷新。
- [x] 保证删除失败时不清缓存、不伪报成功。
- [x] 增加测试，证明删除成功触发刷新，失败不触发。

## 二、分类统计与删除边界

- [x] 将分类页观察统计改为复用统一的“原生优先”算法。
- [x] 对查询范围两端进行裁剪，避免跨边界记录多计时长。
- [x] 保留确定性的应用名与最近出现时间选择规则。
- [x] 区分原生、外部和混合来源数据的底层所有权，但不制造不同的应用管理体验。
- [x] “删除应用记录”分别在原生表和外部表中删除所选应用，不能波及其他应用或来源记录。
- [x] 删除外部记录后同步批次数量，并清理已经没有记录的空批次。
- [x] 增加重叠、跨边界、外部独占、混合来源和删除隔离测试。

## 三、History 汇总与时间线粒度

- [x] 时间线继续只显示精确会话，不伪造小时汇总的顺序和位置。
- [x] 小时汇总对应应用能够正常加载图标与分类展示。
- [x] 增加测试，证明汇总纳入小时数据且时间线不纳入。
