# 外部导入备份隔离与原生优先执行单

状态：已完成并归档（2026-07-18）

## 第一性原理不变量

- [x] `sessions` 及其标题样本只代表 Patina 原生采集事实。
- [x] `import_exact_sessions`、`import_time_buckets` 只代表外部导入事实。
- [x] 备份可以同时携带两类事实，但恢复不得跨所有权边界转换记录。
- [x] 覆盖恢复同时替换原生区和外部区；旧备份没有外部区时视为空外部区。
- [x] 合并恢复只做原生与原生、外部与外部的去重合并。
- [x] 读取时优先采用 Patina 原生事实；外部精确记录只补原生未覆盖区间。
- [x] 外部小时汇总只占用原生和外部精确记录之后的剩余小时容量。
- [x] 分类与应用映射属于 Patina 全局设置，按现有设置恢复语义处理，不随导入批次删除。

## 阶段 A：备份合同测试

- [x] 快照清单分别记录外部批次、外部精确记录和外部小时汇总数量。
- [x] 恢复预览 IPC 严格解析并展示三项外部数量。
- [x] 旧结构化备份的三项外部数量固定为零。
- [x] 静态合同禁止外部备份 owner 把 `import_*` 写入 `sessions`。

## 阶段 B：恢复所有权实现

- [x] v2 覆盖恢复继续整体替换数据库，并在快照回环测试中验证两类外部记录仍位于各自表中。
- [x] v2 合并恢复把候选数据库的外部批次与记录合并进当前外部区。
- [x] 外部合并按记录类型与指纹去重；没有新增记录时不创建空批次。
- [x] 批次 ID 冲突时生成新的内部 ID，不覆盖当前批次。
- [x] 原生合并、外部合并和设置合并在一个事务中提交或整体回滚。
- [x] 旧结构化备份执行覆盖恢复时清空当前外部区；合并恢复时保留当前外部区。

## 阶段 C：原生优先读取

- [x] History 中外部精确记录按原生覆盖区间切分，只显示未覆盖片段。
- [x] Dashboard 和 Data 的聚合读取使用同一优先规则；Classification 只读取应用与全局映射，不另算时间。
- [x] 外部小时汇总在同一小时剩余容量内按原始时长比例分配。
- [x] 原生记录、外部精确记录和小时汇总均保持原库存储不变。
- [x] 精确区间裁剪采用扫描线与最小堆；10,000 条重叠记录的聚焦测试通过。
