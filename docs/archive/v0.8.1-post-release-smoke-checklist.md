## 9. 结果判定

### 通过条件

- Release 产物完整。
- 干净安装或覆盖安装至少一条路径通过。
- 主窗口、托盘、Widget、核心页面能正常使用。
- 没有数据丢失、无法启动、安装失败、版本错误、updater artifact 错误。

### 需要补发 `0.8.2` 的条件

- 安装包无法安装或启动。
- About / updater 显示版本错误。
- `latest.json` 指向错误版本或错误附件。
- 旧数据升级后不可读。
- 备份 / 恢复出现高风险错误。
- 主路径页面无法打开。

### 可延后到后续版本的条件

- 非阻断文案问题。
- 低频页面轻微布局瑕疵。
- 不影响数据安全和启动使用的次要体验问题。
