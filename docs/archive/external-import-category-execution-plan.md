# 外部导入分类执行方案

状态：已完成并归档

## 规则

- [x] 分类判断以规范化后的 `exe_name` 为单位。
- [x] 无有效分类时保持未分类。
- [x] 只有一个唯一分类时创建或复用 Patina 分类并归类。
- [x] 两个及以上不同分类时保持未分类，不按频次或顺序猜测。
- [x] 已有用户手动分类优先，不被导入覆盖。
- [x] 删除导入批次不删除分类，也不改变 Patina 本体记录。
- [x] 不新增数据库迁移；复用现有 `settings` 分类模型。

## 第一性原理结果

1. 导入数据和由本次导入生成的分类设置必须同时成功或同时失败，因此二者在同一 SQLite 事务内提交；不采用事后删除批次的补偿式回滚。
2. 分类属于 Patina 的全局用户设置，而不是导入批次的附属物；删除批次只删除隔离的外部记录，保留分类定义和应用映射。
3. 导入只能陈述 CSV 中存在的事实：零个分类不猜测，一个分类才归类，两个及以上分类明确降级为未分类。
4. 分类候选按规范化可执行文件聚合，空值、`未知` 和 `unknown` 不构成分类；英文大小写重复不构成两个分类。
5. 已有用户分类比外部导入更高权重，导入只补全没有手动分类的应用。
