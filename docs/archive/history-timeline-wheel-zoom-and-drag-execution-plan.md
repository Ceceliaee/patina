# History 时间轴缩放、平移与分轨执行记录

> 状态：已完成并归档  
> 完成日期：2026-07-15  
> 对应工作项：`支持时间轴滚轮缩放与鼠标拖动`  
> Project 最后核对状态：`In progress`，仍需维护者手动拖到 `Done`

## 1. 最终结论

本任务沿用原有“当前时间窗口”交互，不引入全天总览或额外选择框：

- 上方主时间轴直接显示当前时间窗口；
- 普通滚轮仍以指针位置为锚点连续缩放；
- `Shift + 滚轮`、横向滚轮和主键拖动仍平移当前窗口；
- 原来的 `24h / 12h / 8h / 4h / 1h` 五个预设按钮，替换为项目已有样式的步进滑杆；
- 滑杆范围为 1～24 小时，默认 4 小时，直接拖动步长为 0.2 小时；两侧按钮始终跳到相邻整数小时；
- 普通滚轮每个有效滚动事件固定缩放 0.2 小时（12 分钟）；
- 下方应用或分类分轨与上方共享当前窗口，固定显示五行高度，更多行在内部滚动；
- 时间段只保留悬浮信息，不提供无意义的点击、焦点或按压动效。

## 2. 第一性原理

用户真正控制的是同一个时间窗口的两个维度：

1. 窗口长度：决定查看细节的尺度；
2. 窗口位置：决定查看一天中的哪一段。

滚轮缩放适合连续探索，滑杆适合显式设定大致小时数；二者应控制同一个状态，而不是创造“总览窗口”和“选择窗口”两套状态。分轨只负责把同一个窗口按应用或分类拆开解释。

必须保持的不变量：

- [x] 时间窗口始终位于选中日期的本地 00:00～24:00 内。
- [x] 上方主轨与下方所有分轨共享同一 `startMs/endMs`。
- [x] 缩放改变窗口长度，并尽量保持指针锚定时刻在屏幕中的位置。
- [x] 平移只改变窗口位置，不改变窗口长度。
- [x] 到达日期边界后钳制，不制造越界状态。
- [x] 应用/分类切换只改变分组，不改变当前窗口。
- [x] 只持久化窗口长度，不持久化窗口起点。

## 3. 可勾选执行方案

### 3.1 小时滑杆复用

- [x] 将误放在 Settings 下的通用 `SettingsStepperSlider` 提升为 `shared/components/QuietStepperSlider.tsx`。
- [x] Settings 继续使用同一个 shared 组件，保持原有分钟步进行为。
- [x] History 直接复用该组件，不复制结构、样式或键盘逻辑。
- [x] History 配置 `min=1`、`max=24`、`step=0.2` 和整数吸附按钮。
- [x] 小数值点击 `+` 跳到下一个整数，点击 `−` 跳到上一个整数；整数值正常加减 1。
- [x] 首次使用默认 4 小时。
- [x] 显示当前小时值，并使用等宽数字避免布局跳动。
- [x] 在 1 小时与 24 小时边界禁用对应按钮。
- [x] 保留原生 range 的键盘操作、焦点环与可访问名称。
- [x] 非法、空白和越界存储值回退到 4 小时。
- [x] 连续滚轮缩放产生的小数小时不被强制取整。

### 3.2 连续缩放

- [x] 普通纵向滚轮执行连续缩放，而不是平移。
- [x] 每个有效滚动事件固定改变 0.2 小时，不随当前窗口大小改变步幅。
- [x] 缩放以指针在轨道内的横向比例为锚点。
- [x] 缩放范围钳制在 1～24 小时。
- [x] 高频 wheel 更新通过 `requestAnimationFrame` 合并。
- [x] 上方主轨、小时滑杆和下方分轨同步更新。
- [x] 只有视口实际变化时才阻止默认滚动。

### 3.3 平移与拖动

- [x] `Shift + 纵向滚轮`执行横向平移。
- [x] 触控板或鼠标横向滚轮执行横向平移。
- [x] 主键拖动超过 5px 阈值后才进入 dragging 状态。
- [x] 主轨在 capture 阶段取得 pointer capture；颜色段的悬浮框只响应 hover，不再处理主轨上的 pointerdown，避免悬浮状态更新与长按拖动竞争。
- [x] 从主轨颜色条中心按住并停留后，拖出颜色条范围仍持续平移；分轨交互保持不变。
- [x] 拖动使用 pointer capture，移出轨道后松开仍能清理。
- [x] `pointerup`、`pointercancel`、失焦和关闭弹窗均清理交互状态。
- [x] stale `lostpointercapture` 不会取消已经重新捕获的拖动。
- [x] 平移后窗口长度和滑杆数值保持不变。
- [x] 到达日期边界后停止继续移动。

### 3.4 分轨区域

- [x] 应用模式按应用聚合当前窗口内的时间段。
- [x] 分类模式按分类聚合当前窗口内的时间段。
- [x] 轨道按当前窗口内时长降序排列，相同时使用稳定键排序。
- [x] 应用模式优先显示应用图标；无图标及分类模式显示语义色点。
- [x] 每行显示名称、轨道和当前窗口内时长。
- [x] 分轨不重复绘制横轴，所有行共享上方当前窗口坐标。
- [x] 分轨容器固定为 `5 × 50px = 250px`。
- [x] 不足五行和空状态仍保留五行高度，避免弹窗跳动。
- [x] 超过五行时只在分轨区域内部纵向滚动。
- [x] 下方纵向滚动不触发上方时间窗口缩放或平移。

### 3.5 时间段反馈

- [x] 时间段保留指针悬浮信息框。
- [x] 悬浮信息框复用 `QuietTooltip` 门户渲染到 `document.body`，不受五行滚动容器的 overflow 裁剪。
- [x] 靠近视口边缘时自动翻转并限制在可视区域内。
- [x] 时间段不可点击，不进入 Tab 顺序。
- [x] 移除 hover 增高、active 压缩和 focus 强调等可点击暗示。
- [x] dragging 期间立即关闭并抑制悬浮信息框。
